# Codex MCP 服务器设计记录

本文件为中文版本，规范文档以 `mcp-server-discussion.md`（英文）为准。

## 重点

- 这是持续更新的设计日志，而非一次性定稿。
- 目标是将模糊需求结构化，提升执行可靠性。
- `generate_plan` 前必须完成 council 共识。
- UI/UX 任务在满足条件时必须通过 `visual_review`。
- 最终完成必须满足 `record_user_feedback(approved=true)`。
- 团队角色可按会话动态配置（`council_configure_team`）。

详细内容请以英文版为准。
